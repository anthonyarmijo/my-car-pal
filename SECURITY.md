# Security Policy

## Supported Versions

The public core is currently in pre-1.0 active development. Security fixes are expected to target the current default development branch until a stable release policy is published.

## Reporting A Vulnerability

Please do not open a public issue for suspected vulnerabilities.

Until a dedicated disclosure channel is finalized, report security concerns by emailing `support@mycarpal.app` with:

- a short description of the issue
- affected routes, files, or configuration when known
- reproduction steps or proof-of-concept details
- whether any user data may be affected

We will acknowledge reports as soon as practical and coordinate next steps privately. This placeholder policy should be replaced with the final disclosure address and response targets before the repository is made public.

## Scope

Security-sensitive areas include authentication, session handling, user data isolation, uploaded documents, storage adapters, environment configuration, and database migrations.

Hosted-service infrastructure, billing providers, production deployment configuration, and customer operations are outside this repository's security-reporting scope.
